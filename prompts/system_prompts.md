# 弥娅系统提示词库

本文档存储弥娅的各种系统提示词模板，可直接复制到 `config/.env` 中使用。

---

## 📋 使用方法

1. 选择适合的提示词模板
2. 复制整个提示词内容
3. 编辑 `config/.env` 文件
4. 找到 `SYSTEM_PROMPT` 配置项
5. 粘贴提示词内容（保持单行或多行格式）
6. 重启弥娅系统

## 🎯 自定义提示词指南

### 提示词结构

一个完整的系统提示词通常包含：

1. **身份定义** - 你是谁
2. **核心能力** - 你擅长什么
3. **回应风格** - 如何回答问题
4. **限制条件** - 不能做什么
5. **人格特质** - 五维人格向量

### 示例模板

```
你是[名称]，一个[角色]。你擅长[核心能力列表]。

你的五维人格特质：
- 温暖度：[数值] - [描述]
- 逻辑性：[数值] - [描述]
- 创造力：[数值] - [描述]
- 同理心：[数值] - [描述]
- 韧性：[数值] - [描述]

你的回应应该：[回应风格]。
```

### 编写技巧

1. **明确角色定位**
   - ✅ 你是一个专业的Python开发工程师
   - ❌ 你会编程

2. **列出核心能力**
   - ✅ 擅长Python、JavaScript、代码调试、性能优化
   - ❌ 什么都会

3. **定义回应风格**
   - ✅ 回答应该简洁、准确、包含代码示例
   - ❌ 回答要短

4. **添加限制条件**
   - ✅ 不要输出超过100行代码，优先使用标准库
   - ❌ 别写太长

5. **设定人格数值**
   - 温暖度：0.0-1.0，越高越友好
   - 逻辑性：0.0-1.0，越高越严谨
   - 创造力：0.0-1.0，越高越创新
   - 同理心：0.0-1.0，越高越善解人意
   - 韧性：0.0-1.0，越高越抗压

## 📝 配置方法

### 方法1：直接编辑 .env 文件

```batch
notepad config\.env
```

找到 `SYSTEM_PROMPT` 配置项，替换为你选择的提示词：

```env
SYSTEM_PROMPT=你是弥娅（Miya），一个专业的编程助手...
```

**注意**：
- 如果提示词包含换行，需要用 `\n` 表示
- 或者保持单行，用空格分隔句子

### 方法2：使用JSON配置文件（推荐）

创建 `prompts/custom_prompt.json`：

```json
{
  "system_prompt": "你是弥娅（Miya），一个专业的编程助手...",
  "user_prompt_template": "用户输入：{user_input}",
  "personality_context_enabled": true,
  "memory_context_enabled": true,
  "memory_context_max_count": 5
}
```

在代码中加载：

```python
from core import PromptManager
from pathlib import Path

pm = PromptManager()
pm.load_from_json(Path('prompts/custom_prompt.json'))
```

## 🔄 管理多个提示词

### 提示词变体

你可以为不同场景创建多个提示词变体：

| 文件名 | 场景 |
|--------|------|
| `prompts/standard.json` | 标准助手 |
| `prompts/developer.json` | 编程助手 |
| `prompts/writer.json` | 写作助手 |
| `prompts/tutor.json` | 学习导师 |
| `prompts/creative.json` | 创意伙伴 |

### 动态切换

```python
from core import PromptManager
from pathlib import Path

pm = PromptManager()

# 加载编程助手提示词
pm.load_from_json(Path('prompts/developer.json'))

# ... 使用编程助手模式 ...

# 切换到写作助手模式
pm.load_from_json(Path('prompts/writer.json'))

# ... 使用写作助手模式 ...
```

## 🧪 测试提示词

### 测试步骤

1. 设置提示词
2. 重启弥娅
3. 发送测试消息
4. 观察回应是否符合预期

### 测试问题示例

- "你好，请介绍一下自己"
- "什么是递归？请用Python举例"
- "帮我写一首关于春天的诗"
- "我想学习编程，有什么建议？"

### 评估标准

- ✅ 回应符合角色定位
- ✅ 语言风格一致
- ✅ 人格特质体现
- ✅ 回答质量高
- ✅ 没有偏离设定
