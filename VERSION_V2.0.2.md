## 版本特性

### 1. 表情包功能修复 ✅ 已修复
**问题**: 弥娅无法发送data目录中的本地表情包，只能发送内置QQ表情

**解决方案**:
1. **增强表情包发送系统** (`webnet/qq/message_handler.py`):
   - 修改 `_send_emoji_response` 方法，优先查找本地表情包
   - 新增 `_send_local_emoji` 方法，支持从data目录查找图片文件
   - 实现智能回退机制：本地文件 → QQ内置表情 → 文本表情

2. **表情包查找逻辑**:
   - 优先检查 `./data/emojis/` 目录
   - 支持多种图片格式（.png, .jpg, .jpeg, .gif）
   - 大小写不敏感匹配
   - 关键词模糊匹配（如"开心"匹配"开心.jpg"）

### 2. 定时功能修复 ✅ 已修复
**问题**: 定时/提醒功能失效，无法处理"一分钟后提醒我喝水"等请求

**解决方案**:
1. **增强定时任务检测** (`hub/decision_hub.py`):
   - 新增 `_detect_and_process_timer_task` 方法
   - 支持多种定时关键词（定时、提醒、时间、之后、分钟、小时）
   - 集成中文时间解析（一分钟、五分钟、半小时等）

2. **定时任务处理流程**:
   - 在 `process_perception_cross_platform` 中集成定时检测
   - 通过ToolNet的 `create_timer_task` 工具创建定时任务
   - 添加详细的执行日志，便于调试

### 3. 诊断与调试改进 ✅ 已优化
1. **增强日志系统**:
   - 在所有关键路径添加详细日志
   - 区分调试、信息和错误级别
   - 记录表情包查找和定时任务创建的完整过程

2. **错误处理优化**:
   - 本地表情包查找失败时优雅回退
   - 定时任务创建失败时提供明确错误信息
   - 增加异常捕获和用户友好提示

## 技术实现细节

### 表情包系统架构
```
用户请求表情 → 优先查找本地文件 → 文件存在？ → 发送图片
                                  ↓ 文件不存在
                            查找QQ内置表情 → 表情存在？ → 发送QQ表情
                                              ↓ 不存在
                                          发送文本表情
```

### 定时任务检测流程
```
用户消息 → 包含定时关键词？ → 是 → 解析时间参数
                    ↓ 否                     ↓
               跳过检测                调用ToolNet创建定时任务
                                          ↓
                                      返回执行结果和确认消息
```

## 配置说明

### 表情包目录配置
确保 `config/qq_config.yaml` 中配置了正确的表情包路径：
```yaml
emoji:
  directory: "./data/emojis"
```

如果不存在该目录，系统会自动创建：
```python
# 自动创建表情包目录（如果不存在）
os.makedirs("data/emojis", exist_ok=True)
```

### 定时任务关键词配置
系统自动识别以下关键词：
- **定时相关**: 定时、提醒、记得、时间
- **时间单位**: 分钟、小时、秒、之后、以后
- **相对时间**: 一、两、三、五、十、半、个

## 使用方法

### 1. 发送表情包
```plaintext
用户: 发送开心表情
Miya: [发送data/emojis/开心.jpg]
      或[发送QQ表情]
      或[发送文本表情]
```

### 2. 创建定时任务
```plaintext
用户: 一分钟后提醒我喝水
Miya: [检测到定时请求]
      [创建定时任务: 60秒后提醒"喝水"]
      [返回确认消息]
```

### 3. 混合请求
```plaintext
用户: 发送表情然后五分钟后提醒我开会
Miya: [发送表情包]
      [创建定时任务: 300秒后提醒"开会"]
```
